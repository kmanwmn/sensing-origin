## 项目架构概览

```
前端（Vercel）             后端（腾讯云服务器 124.223.88.48）
┌──────────────────┐       ┌──────────────────────────────────────┐
│  www.sensing-    │  ──→  │  Nginx（443/8443）                   │
│  origin.cn       │       │    ├─ /vod/    → 视频文件直连         │
│  (Vercel CDN)    │       │    ├─ /hls/    → HLS 直播切片         │
│                  │       │    └─ /api/*   → 反向代理到 5000      │
│  静态页面:       │       │       └─ HttpApiServer（端口 5000）    │
│  index.html      │       │  NetDisk TCP 服务器（端口 8002）       │
│  news.html       │       │  Nginx-RTMP 流媒体（端口 1935）        │
│  tech.html       │       └──────────────────────────────────────┘
│  图片/           │
└──────────────────┘
```

- **前端**：Vercel 托管静态页面
- **API**：Nginx 反向代理到 HttpApiServer（端口 5000）
- **视频流**：Nginx 直连 `/vod/` 目录（扁平路径 `<用户名>_<文件名>`）
- **RTMP 流媒体**：端口 1935
- **数据服务**：NetDisk TCP 服务器（端口 8002）
- **数据库**：MySQL（NetDisk 库）

### ⚠️ 域名别名的重要注意事项

**问题现象**：代码推送到 GitHub 后，Vercel 自动部署了新版本，但用域名打开还是旧版本。

**原因**：Vercel 项目默认绑定的 production 域名（`xxx.vercel.app`）会自动跟随最新部署。但如果你用 `vercel alias set` 手动设置过自定义域名别名，这个别名是**锁定到某个具体部署 URL 的**，后续推送的新部署不会自动更新它。

**解决方式**：

```bash
# 1. 查看最新部署 URL
npx vercel list

# 2. 删除旧的域名别名
npx vercel alias rm www.sensing-origin.cn

# 3. 重新设置为最新部署
npx vercel alias set <最新部署URL> www.sensing-origin.cn
```

**最佳实践**：
- 首次绑定时，在 Vercel 项目设置（Dashboard → Project → Domains）中添加域名，让系统自动管理关联
- 避免使用 `vercel alias set` 手动设置，除非有特殊需要（如回滚到某个旧版本）
- 如果确实用了手动别名，每次 `git push` 后记得检查域名是否已更新：`curl -I https://www.sensing-origin.cn/news.html | grep -i etag`
- Vercel 的 Production 部署会自动绑定到项目的 Production domain（`xxx.vercel.app`），这个是实时更新的

### 视频域名配置

`video.sensing-origin.cn` 直接解析到腾讯云服务器 IP，Nginx 处理所有请求：

#### Nginx 配置（/etc/nginx/conf.d/video-ssl.conf）

```
端口 443 + 8443（HTTPS）
├── /vod/  → 视频文件直连（/home/ubuntu/NetDisk/video/flv/）
├── /hls/  → HLS 直播切片（/tmp/hls/）
├── /api/* → 反向代理 → http://127.0.0.1:5000（HttpApiServer）
└── /      → 静态文件（/var/www/html）

端口 80（HTTP）
└── 全部重定向到 HTTPS
```

#### SSL 证书

- 提供商：ZeroSSL
- 域名：`video.sensing-origin.cn`
- 证书路径：`/etc/nginx/ssl/video.sensing-origin.cn.fullchain.crt`
- 密钥路径：`/etc/nginx/ssl/video.sensing-origin.cn.key`
- 有效期：2026-06-03 至 2026-09-01

## 四、API 地址配置

### 前端 JS 中的 API 地址

两个文件中需要配置 `API_BASE_URL`：

| 文件 | 变量 |
|------|------|
| `news.html` | `var API_BASE_URL = 'https://video.sensing-origin.cn:8443'` |
| `tech.html` | `var API_BASE_URL = 'https://video.sensing-origin.cn:8443'` |

当部署到 Vercel（`www.sensing-origin.cn`）时，浏览器从 Vercel 加载页面，JS 通过 HTTPS 跨域请求 `video.sensing-origin.cn:8443` 上的 API。

CORS 头由 Nginx 添加：
```
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET,POST,PUT,DELETE,OPTIONS
Access-Control-Allow-Headers: Content-Type,Authorization
```

### 修改 API 地址的步骤

1. 修改 `news.html` 和 `tech.html` 中的 `API_BASE_URL`
2. `git add . && git commit -m "修改说明" && git push origin main`
3. 等待 Vercel 自动部署（约 10-15 秒）
4. 验证：`curl -s https://www.sensing-origin.cn/news.html | grep API_BASE_URL`

## 七、常见问题

### Q: 浏览器打开显示"无法连接服务器"

可能原因：
- 前端 `API_BASE_URL` 配置的端口在腾讯云安全组未开放
- DNS 未生效
- Nginx 未运行或配置错误
- HttpApiServer 未运行

解决办法：
1. 检查 API 是否可达：浏览器直接打开 `https://video.sensing-origin.cn:8443/api/news`
2. 检查腾讯云安全组是否放行了相应端口
3. 检查 Nginx 状态：`systemctl status nginx`
4. 检查后端进程：`pgrep -a HttpApiServer`

### Q: 部署后页面没有变化

可能原因：
- Vercel 部署尚未完成
- 手动设置的域名别名指向了旧部署

解决办法：
1. 等待部署完成：`npx vercel list` 查看状态
2. 更新别名：`npx vercel alias set <新部署URL> www.sensing-origin.cn`
3. 或清除浏览器缓存：`Ctrl+Shift+R` 硬刷新

### Q: 需要更换服务器 IP

1. 更新域名 DNS 记录
2. 更新前端 `API_BASE_URL`
3. 提交推送触发 Vercel 部署
4. 重新申请 SSL 证书
